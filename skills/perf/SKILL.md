# 海外思维 LP/TL 转介绍激励计算

从 BI 下载源数据 → 生成业绩明细 → 计算 LP/TL 激励，输出到 `26年Y月海外思维转介绍激励.xlsx`。

## 触发方式

用户说「计算激励」「跑激励」「更新激励」或调用 `/perf`。

## 完整流程

### 第 1 步：下载 BI 源数据

使用 Playwright 自动登录 Smartbi 下载以下 3 个报表到 `激励/` 文件夹：

| 报表 | 搜索关键词 | 筛选条件 |
|------|-----------|----------|
| 销售明细 | `益智海外用户销售明细_转介绍` | 末次渠道时间=当月第一天~最后一天，末次进线渠道一级分类=转介绍，勾选后刷新 |
| 业绩播报 | `转介绍益智业绩播报_LP维度_末次渠道` | 无 |
| LP架构表 | 搜索「架构表」相关报表 | 无 |

下载完成后输出文件列表确认。

### 第 2 步：下载滚动成单（自动）

自动从 BI 下载「小组-滚动成单」，报表为 `益智海外用户销售明细_转介绍`，筛选条件（动态计算）：

| 筛选项 | 值 |
|--------|-----|
| 末次渠道时间开始 | 2020-01-01（固定） |
| 末次渠道时间结束 | 当月最后一天 |
| 首签开始时间 | 当月第一天 |
| 首签结束时间 | 当月最后一天 |
| 末次进线渠道一级分类 | 转介绍 |

下载后自动保存为 `激励/小组-滚动成单.xlsx`。使用 bi_skill 的 `_apply_date_filter_v2` 和 `_apply_multiselect_filter` 进行筛选。

### 第 3 步：生成业绩明细

过滤规则：
- 当月例子 / 重复进线例子：只取 `业绩归属lp_团队` ∈ {港澳益智教学服务区, 美澳益智教学服务区, 台湾益智教学服务区}
- 滚动成单：末次渠道更新月份 ≠ 当月 + `推荐人业绩归属（首消）`= 班主任 + `业绩归属lp_团队` ∈ 上述三区 + 益智海外教学服务区

输出 sheet `业绩明细`，含 10 列：ID(学员ID), 业绩类型, 业绩LP, 业绩小组, 末次渠道更新时间, 约课时间, 到课时间, 成单时间, GMV, 业绩时间。

### 第 4 步：计算 LP 激励

基于业绩明细 + 业绩播报 + LP架构表计算每位 LP 的激励，输出 sheet `LP激励`（31 列）。

### 第 5 步：计算 TL 激励

基于 LP 激励数据 + 业绩播报计算每位 TL 的激励，输出 sheet `TL业绩汇总` 和 `TL激励`。

### 第 5 步：计算 LP经理激励

基于 LP激励数据 + TL业绩数据计算TL_E的激励（门槛：GMV≥85%）。

输出 sheet `LP经理激励`。

### 第 6 步：计算 对接负责人激励

基于 TL业绩数据计算TL_G的激励（海外团队例子+港澳到课+5.17进度）。

输出 sheet `对接负责人激励`。

### 第 7 步：计算 拓科激励

基于拓科数据源计算TL_C和各小组TL的拓科激励（需额外拓科数据源）。

输出 sheet `拓科激励`。

激励规则来自 `激励/26年Y月转拓激励方案.xlsx`。核心逻辑：

### 打卡率门槛
- 小组 M1-M3 打卡率 ≥ 小组 M1-M3 目标 → 全组通过
- 否则个人 M1-M3 打卡率 ≥ 阈值（港澳/美澳 45%，台湾 30%）
- 新人免门槛

### 例子绩效（所有 LP）
按例子达成率分档：0~75%=0, 75~90%=5, 90~100%=6, 100~110%=7, ≥110%=8 元/例子

### 非港澳成单激励（阶梯制）
| 档位 | 条件 | 元/单 |
|------|------|-------|
| C | 例子进度 < 90% | 50 |
| B | ≥90% 且 1-2单 | 120 |
| A | ≥90% 且 3-7单 | 150 |
| S | ≥90% 且 8+单 | 180 |

新人：100 元/单，不与阶梯叠加

### 港澳成单 + 到课激励
- 到课激励（按到课数分档）：C=10, B=25, A=40, S=60 元/到课
- 成单激励：仅 A 档（例子达成 ≥ 100%）→ 100 元/单
- 新人：成单 100 元/单 + 到课激励叠加

### 排名奖
- Top 20 按总例子排名，需打卡率达标 + GMV 达成 ≥ 90%
- 第1名 800, 2-3名 600, 4-10名 400, 11-20名 200

### 个人流速奖金池
- 总池 10,000 元，需打卡率达标 + 例子达成 ≥ 60%
- 按个人例子占比分配，单人上限 2,000 元
- 美澳5组不参与

## TL激励规则（2026年5月起）

TL激励包含3个部分，可叠加：

### 1. 例子奖金池（流速激励）
- **奖金池金额**: 3,000元
- **门槛**: 截止5月17日24点，小组例子进度≥60%
- **瓜分方式**: 达成门槛的小组按例子数占比瓜分
- **单人上限**: 1,000元
- **例外**: 美澳5组不参与瓜分

### 2. 到课奖金池
- **奖金池金额**: 5,000元
- **门槛**: 小组到课目标达成100%
- **瓜分方式**: 达成门槛的小组按到课数占比瓜分
- **单人上限**: 2,000元
- **例外**: 美澳5组不参与瓜分

### 3. 转介绍提成
基于**转介绍综合达成率**计算：
```
转介绍综合达成率 = 转介绍leads达成率 × 0.8 + 滚动GMV达成率 × 0.2
```

| 综合达成率 | 提成金额 |
|-----------|---------|
| <80% | 0 |
| 80%≤X<90% | 500 |
| 90%≤X<100% | 2,600 |
| 100%≤X<110% | 3,200 |
| X≥110% | 3,600 |

### 台湾组（TL_I）特殊计算
由于TL_I同时是TL和LP，**TL层级数据要剔除他自己的数据**：
```
台湾组TL例子达成 = 台湾组总例子 - TL_I例子
台湾组TL例子目标 = 台湾组目标 - TL_I目标
台湾组TL到课数 = 台湾组总到课 - TL_I到课
```

## LP经理激励规则（TL_E）

**激励对象**：TL_E（计算欧美澳团队+台湾团队达成，剔除TL_I）

**门槛**：滚动GMV月度完成率 >= 85%（不含国带海，不含勘误）

| 激励类型 | 档位 | 条件 | 金额 |
|:---|:---|:---|:---:|
| **例子流速激励** | A | 5月17日例子达成60% | 2,500 |
| （截止5月17日） | S | 5月17日例子达成65% | 3,000 |
| | S+ | 5月17日例子达成70% | 3,500 |
| **例子达成激励** | B | 全月例子达成95% | 2,000 |
| （全月） | A | 全月例子达成100% | 2,500 |
| | S | 全月例子达成110% | 3,000 |

## 对接负责人激励规则（TL_G）

| 项目 | 条件 | 金额 | 门槛 |
|:---|:---|:---:|:---|
| ①海外团队例子达成 | 例子达成率>=100% | 500 | GMV>=85% |
| ②港澳到课例子达成 | 到课例子达成率>=95% | 1,000 | GMV>=85% |
| ③5月17日例子进度 | 例子进度>=60% | 1,000 | 无 |

## 拓科激励规则（TL_C+各小组TL）

**对接人TL（TL_C）**：
- 14日进度达成55% → 600元
- 全月达成100% → 600元

**各小组TL**：
- 奖金池：2,000元
- 瓜分方式：按小组滚动GMV贡献占比
- 门槛：小组拓美术例子达成100%

```bash
python3 -c "import playwright; import pandas; import openpyxl" 2>&1 || pip3 install playwright pandas openpyxl
```

## 执行命令

```bash
python3 .claude/plugins/incentive-calculator/main.py \
  --month 5 --year 2026 \
  --base-dir /Users/dory/Desktop/claude/激励
```

选项：
- `--month` / `--year`：计算月份（默认当月）
- `--base-dir`：数据文件夹路径
- `--skip-download`：跳过 BI 下载，直接计算（已有所有源文件时使用）
- `--skip-incentive`：只生成业绩明细，不计算激励

## 输出文件

`{base-dir}/26年{year}月海外思维转介绍激励.xlsx`，含 7 个 sheet：
- `业绩明细`：逐行明细（当月例子 / 重复进线例子 / 滚动成单）
- `LP激励`：按 LP 汇总的激励计算结果（31 列）
- `TL业绩汇总`：按 TL 汇总的业绩数据
- `TL激励`：TL 激励计算结果（含例子奖金池、到课奖金池、转介绍提成）
- `LP经理激励`：TL_E激励（门槛：GMV≥85%，含流速激励+达成激励）
- `对接负责人激励`：TL_G激励（海外团队例子+港澳到课+5.17进度）
- `拓科激励`：TL_C+各小组TL拓科激励（需拓科数据源）

## 关键实现细节

- 销售明细 header 在 row 7（0-indexed），数据从 row 8 开始
- 业绩播报为多段结构（海外-整体 / 团队 / 区域 / 小组 / LP / 主管），需按段解析
- LP 筛选：`职位=班主任` 且 `是否在职=在职`（+ TL_I）
- 团队过滤：排除非益智教学服务区的团队（武汉、上海、大连等 LP 团队）
- 到课次数来自销售明细 col 23，按 LP 汇总用于港澳到课激励
- 当月入职判断：入职时间 ≥ 当月 1 日
- TL激励计算时台湾组需剔除TL_I数据
- 到课目标从业绩播报列63（索引63）获取
